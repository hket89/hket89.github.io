# Next.js Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the portfolio site from Vite + React SPA to Next.js 15 App Router with static export, adding SEO (meta tags, Open Graph, JSON-LD) while keeping the site on GitHub Pages at hket89.github.io.

**Architecture:** Next.js App Router with `output: 'export'` generates fully static HTML/CSS/JS into `out/`. A `ThemeRegistry` component handles MUI + Emotion style injection during static generation. GitHub Actions deploys the `out/` directory to the `gh-pages` branch, unchanged from before except the output path.

**Tech Stack:** Next.js 15, React 19, MUI 7, Emotion, TypeScript, pnpm, GitHub Actions, GitHub Pages

`output: 'export'` — write static files to `out/` instead of running a server.

## Task 4: Create ThemeRegistry

**Files:**
- Create: `src/app/theme-registry.tsx`

- [ ] **Step 1: Create src/app/theme-registry.tsx**

```tsx
'use client'

import { useState } from 'react'
import { useServerInsertedHTML } from 'next/navigation'
import createCache from '@emotion/cache'
import { CacheProvider } from '@emotion/react'
import { ThemeProvider, CssBaseline } from '@mui/material'
import { theme } from '../theme'

export function ThemeRegistry({ children }: { children: React.ReactNode }) {
  const [{ cache, flush }] = useState(() => {
    const cache = createCache({ key: 'mui' })
    cache.compat = true
    const prevInsert = cache.insert.bind(cache)
    let inserted: string[] = []
    // eslint-disable-next-line @typescript-eslint/no-explicit-any
    cache.insert = (...args: any[]) => {
      const serialized = args[1]
      if (cache.inserted[serialized.name] === undefined) {
        inserted.push(serialized.name)
      }
      return prevInsert(...args)
    }
    return {
      cache,
      flush() {
        const flushed = inserted
        inserted = []
        return flushed
      },
    }
  })

  useServerInsertedHTML(() => (
    <style
      data-emotion={`${cache.key} ${flush().join(' ')}`}
      dangerouslySetInnerHTML={{ __html: Object.values(cache.inserted).join(' ') }}
    />
  ))

  return (
    <CacheProvider value={cache}>
      <ThemeProvider theme={theme}>
        <CssBaseline />
        {children}
      </ThemeProvider>
    </CacheProvider>
  )
}
```

What this does:
- Creates an Emotion cache with key `'mui'`
- Intercepts every style insertion and records the style names
- `useServerInsertedHTML` flushes those recorded styles into the static HTML `<head>` during `next build`
- Result: styles are in the HTML before JS loads — no flash of unstyled content

## Task 9: Update GitHub Actions deployment

**Files:**
- Modify: `.github/workflows/deploy.yml`

- [ ] **Step 1: Update publish_dir in deploy.yml**

Change line `publish_dir: ./dist` to `publish_dir: ./out`:

```yaml
name: Deploy Portfolio

on:
  push:
    branches:
      - master

env:
  FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: true

permissions:
  contents: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6.0.2

      - name: Install pnpm
        uses: pnpm/action-setup@v6.0.8
        with:
          version: 10

      - name: Setup Node.js
        uses: actions/setup-node@v6.4.0
        with:
          node-version: '24'
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build
        run: pnpm build

      - name: Deploy to gh-pages
        uses: peaceiris/actions-gh-pages@v4.1.0
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./out
```

Note: `peaceiris/actions-gh-pages` automatically adds a `.nojekyll` file to the published branch. This is critical — Next.js assets live inside `_next/` (underscore prefix), which GitHub's Jekyll processor would otherwise silently strip.

## Implementation Notes

- **og.png**: Replace `public/og.png` with a real 1200×630 PNG before or shortly after the first deploy. A screenshot of the built site taken with browser devtools at 1200px width works well.
- **`next-env.d.ts`**: This file is auto-generated the first time you run `next dev` or `next build`. It is already in `.gitignore` — do not commit it.
- **Hydration warning in dev**: MUI's ThemeProvider may emit a minor hydration mismatch warning in `next dev` mode. This is a known dev-mode artefact and does not appear in the static build output.
- **`@emotion/cache` version**: Must match the `@emotion/react` major version (both `^11.x`). The `package.json` above pins both to `^11.14.0`.
